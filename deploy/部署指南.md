# PDFMaster 云服务器部署指南

## 一、购买云服务器（腾讯云轻量应用服务器）

### 推荐配置
- **平台**: 腾讯云轻量应用服务器 (Lighthouse)
- **系统**: Ubuntu 22.04 LTS
- **配置**: 2核CPU / 4GB内存 / 60GB SSD / 4M带宽 (约 50-70元/月)
- **购买地址**: https://console.cloud.tencent.com/lighthouse

### 购买后操作
1. 在控制台找到服务器的 **公网IP**
2. 重置密码，记录 root 密码
3. 防火墙规则中放行 **80端口** 和 **5700端口** (TCP)

---

## 二、上传项目到服务器

### 方法A: 用 scp 上传 (推荐)

在你的 Windows 电脑上打开终端 (PowerShell/Git Bash)：

```bash
# 将项目打包上传 (排除 venv 和上传文件)
cd D:\桌面\迅读\PDFMaster
tar -czf pdfmaster.tar.gz --exclude=venv --exclude=uploads --exclude=output --exclude=__pycache__ backend frontend requirements.txt Dockerfile docker-compose.yml .dockerignore deploy

# 上传到服务器 (替换 YOUR_IP 为服务器公网IP)
scp pdfmaster.tar.gz root@YOUR_IP:/root/

# SSH 登录服务器
ssh root@YOUR_IP
```

### 方法B: 用 Git (如果有代码仓库)

```bash
ssh root@YOUR_IP
apt update && apt install -y git
git clone https://your-repo-url /opt/pdfmaster
```

---

## 三、部署 (二选一)

### 方案A: Docker 部署 (推荐，最简单)

```bash
# 登录服务器后执行
cd /root
tar -xzf pdfmaster.tar.gz -C /opt/pdfmaster  # 如果用方法A上传
cd /opt/pdfmaster

# 运行一键部署脚本
bash deploy/deploy.sh
```

或者手动执行：
```bash
# 安装 Docker
curl -fsSL https://get.docker.com | sh
systemctl start docker && systemctl enable docker

# 构建并启动
docker compose up -d --build

# 查看状态
docker compose ps
docker compose logs -f
```

### 方案B: 直接部署 (无需 Docker)

```bash
cd /root
tar -xzf pdfmaster.tar.gz -C /opt/pdfmaster
cd /opt/pdfmaster

# 运行直接部署脚本
bash deploy/deploy-direct.sh
```

---

## 四、验证部署

部署完成后，在浏览器中访问：

```
http://你的服务器公网IP
```

如果看到 PDFMaster 主界面，说明部署成功！

> 如果用的是 5700 端口直接访问：`http://你的服务器公网IP:5700`

## 五、绑定域名 (可选)

### 1. 域名解析
在域名服务商处添加 A 记录，指向服务器公网IP。

### 2. 安装 SSL 证书 (HTTPS)
```bash
apt install -y certbot python3-certbot-nginx
certbot --nginx -d your-domain.com
```

### 3. 修改 Nginx 配置
编辑 `/etc/nginx/sites-available/pdfmaster`，将 `server_name` 改为你的域名：
```
server_name your-domain.com;
```
然后重启 Nginx：
```bash
systemctl restart nginx
```

## 七、注意事项

1. **文件清理**: uploads/ 和 output/ 目录会随使用增长，建议设置定时清理：
   ```bash
   # 添加 crontab，每天凌晨3点清理7天前的临时文件
   echo "0 3 * * * find /opt/pdfmaster/uploads /opt/pdfmaster/output -mtime +7 -delete" | crontab -
   ```

2. **内存管理**: PDF处理是内存密集型操作，建议至少 4GB 内存。如内存不足，可在 docker-compose.yml 中调大内存限制。

3. **OCR功能**: 已安装 Tesseract OCR + 中文语言包，开箱即用。

4. **安全建议**:
   - 生产环境建议添加认证机制
   - 使用 Nginx 反向代理而非直接暴露 5700 端口
   - 配置 HTTPS 证书
   - 定期更新系统补丁

5. **带宽要求**: PDF文件较大，建议至少 4M 带宽。如用户较多，建议升级到 8M+。
